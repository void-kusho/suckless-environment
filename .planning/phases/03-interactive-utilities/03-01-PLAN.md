**Includes and structure (per D-01 through D-07):**

**Static helper: `confirm(const char *prompt, char *const extra_argv[])`**
- Opens a second dmenu with the given prompt (per D-05)
- Writes two items: "no" then "yes" (in that order -- "no" is default)
- Reads selection, returns 1 if "yes", 0 otherwise (including cancel/Escape)
- Frees the selection string

**Static helper: `action_lock(void)`** (per D-02, D-03, D-04)
- Check `getenv("DISPLAY")` is non-NULL. If NULL, `warn("DISPLAY not set")` and return (per D-03)
- Check for duplicate: `exec_wait({"pgrep", "-f", "betterlockscreen", NULL})`. If exit status is 0 (process found), return silently (per D-04). Use `-f` not `-x` because betterlockscreen is 16 chars, exceeding Linux's 15-char comm field (per Pitfall 2 in RESEARCH.md)
- Spawn lock: `exec_detach({"betterlockscreen", "-l", "--display", "1", "--blur", "0.8", NULL})` (per D-02)

**Static helper: `action_logout(char *const extra_argv[])`** (per D-05, D-06)
- Call `confirm("logout?", extra_argv)`. If not confirmed, return.
- Kill daemons in order (per D-06): slstatus, dmenu-clipd, dunst, then dwm. For each:
  `exec_wait({"pkill", "-x", daemon_name, NULL})`
- dwm is killed LAST because it is the session leader -- when dwm exits, X session tears down (per Pitfall 7)

**Static helper: `action_reboot(char *const extra_argv[])`** (per D-05, D-07)
- Call `confirm("reboot?", extra_argv)`. If not confirmed, return.
- `exec_detach({"systemctl", "reboot", NULL})` -- use exec_detach because systemctl reboot does not return (per Pitfall 3)

**Static helper: `action_shutdown(char *const extra_argv[])`** (per D-05, D-07)
- Call `confirm("shutdown?", extra_argv)`. If not confirmed, return.
- `exec_detach({"systemctl", "poweroff", NULL})` -- use exec_detach (per Pitfall 3)

**main(int argc, char *argv[]):**
- Set `argv0 = argv[0]`, call `setup_sigchld()`
- Open dmenu: `dmenu_open("session", argv + 1)` -- argv+1 forwards extra args (per D-01)
- Write 4 items in order: "logout", "lock", "reboot", "shutdown" (matching existing shell script menu order per specifics)
- Read selection via `dmenu_read`. If NULL, exit 0 (user cancelled)
- Dispatch: strcmp against each action name, call corresponding action_* function. Pass `argv + 1` to action_logout/action_reboot/action_shutdown for confirmation dialog forwarding
- Free selection, return 0

**Security considerations:**
- DISPLAY validation before spawning X11-dependent betterlockscreen (per D-03)
- No shell interpretation: all process spawning via fork+execvp (never system/popen)
- pgrep -f used for duplicate detection, safe from C per RESEARCH.md Pitfall 1 analysis
- systemctl reboot/poweroff: no user-controlled arguments reach these commands

**Do NOT:**
- Use config.def.h/config.h (no configurable values per RESEARCH.md discretion recommendation)
- Use system() or popen() (CERT C ENV33-C)
- Use exec_wait for systemctl reboot/poweroff (they don't return -- per Pitfall 3)

**Part A: Create `utils/dmenu-cpupower/dmenu-cpupower.c`** (per D-08 through D-12)

**Includes:**

**Static helper: `capture_stdout(const char *const argv[], char *buf, size_t bufsz)`**
- Create pipe, fork child, dup2 stdout to pipe write end, execvp (per Pattern 3 in RESEARCH.md)
- Parent: close write end, read from pipe into buf (up to bufsz-1), close read end, waitpid
- Strip trailing newline. Return 0 on success, -1 on failure.
- This is needed to capture `powerprofilesctl get` output (per D-08)

**main(int argc, char *argv[]):**
- Set `argv0 = argv[0]`, call `setup_sigchld()`
- Capture current profile: `capture_stdout({"powerprofilesctl", "get", NULL}, current, sizeof(current))`
  - If capture fails, set current to "unknown" (per RESEARCH.md discretion recommendation)
- Build prompt: `snprintf(prompt, sizeof(prompt), "cpu: %s", current)` (per D-11)
- Open dmenu: `dmenu_open(prompt, argv + 1)` -- forward extra args (per D-12)
- Write 3 items: "performance", "balanced", "power-saver" (hardcoded per D-09)
- Read selection. If NULL, exit 0.
- Validate selection: strcmp against the 3 valid profile names. If no match (user typed custom text), exit 1. This prevents arbitrary strings being passed to powerprofilesctl (command injection defense).
- Set profile: `exec_wait({"powerprofilesctl", "set", selection, NULL})` (per D-10)
  - If exec_wait returns non-zero, `warn("powerprofilesctl set failed")` (non-fatal per discretion)
- Free selection, return 0

**Security considerations:**
- Profile name validation: only accept exact matches against "performance", "balanced", "power-saver" before passing to exec. Prevents command injection via crafted dmenu input (per security_enforcement: exec args injection).
- No shell interpretation: fork+execvp only.
- capture_stdout properly closes pipe ends in all code paths.

**Do NOT:**
- Use config.def.h/config.h (no configurable values per RESEARCH.md)
- Use cpupower/pkexec (per D-08/D-10 -- powerprofilesctl uses D-Bus, no privilege escalation)
- Pass unvalidated user input to powerprofilesctl set

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dmenu user input -> exec args | User selection from dmenu passed to exec calls |
| environment -> betterlockscreen | DISPLAY env var read and relied upon for X11 lock |
| external binary stdout -> prompt | powerprofilesctl output used in dmenu prompt string |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | dmenu-cpupower: profile name in exec args | mitigate | Validate selection against hardcoded whitelist ("performance", "balanced", "power-saver") before passing to powerprofilesctl set. Reject any other input. |
| T-03-02 | Elevation of Privilege | dmenu-session: systemctl reboot/poweroff | accept | systemctl reboot/poweroff require appropriate polkit/logind permissions. No privilege escalation in our code -- systemd handles authorization. |
| T-03-03 | Spoofing | dmenu-session: DISPLAY env injection | mitigate | Check getenv("DISPLAY") is non-NULL before spawning betterlockscreen. Betterlockscreen itself validates DISPLAY internally. |
| T-03-04 | Denial of Service | dmenu-session: duplicate betterlockscreen spawn | mitigate | pgrep -f check before spawning. If already running, exit silently. |
| T-03-05 | Information Disclosure | dmenu-cpupower: powerprofilesctl output in prompt | accept | Current power profile name is not sensitive information. Displayed in dmenu prompt which is already visible to the user. |
</threat_model>
