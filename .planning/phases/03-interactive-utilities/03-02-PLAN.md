**Struct definition:**
```c
struct entry {
    char hash[NAME_MAX + 1];          /* Hash filename from cache dir */
    char preview[MAX_PREVIEW + 1];    /* Truncated preview text */
    char path[PATH_MAX];              /* Full path to cache file */
    time_t mtime;
};
```

**Static helper: `cmp_mtime_desc(const void *a, const void *b)`** (per D-14)
- Compare mtime descending: newer entries first
- Return -1 if a is newer, 1 if b is newer, 0 if equal

**Static helper: `build_preview(const char *path, char *preview, size_t prevsz)`** (per D-15)
- Open file with fopen, fread up to prevsz-1 bytes
- Replace every '\n' and '\t' with ' ' in the buffer (newlines to spaces per D-15)
- Null-terminate. Close file.
- If file is empty or unreadable, set preview to empty string

**Static helper: `restore_clipboard(const char *filepath)`** (per D-17)
- fork() child process
- In child: open filepath as O_RDONLY, dup2 to STDIN_FILENO, execlp("xclip", "xclip", "-selection", "clipboard", NULL)
- Parent: waitpid for child completion
- This feeds the full file content to xclip's stdin, restoring the complete clipboard entry (not just the preview)

**Static helper: `get_cache_dir(char *buf, size_t bufsz)`** (per D-13)
- Check `getenv("XDG_CACHE_HOME")`. If non-NULL and non-empty, use it (per Pitfall 5 in RESEARCH.md)
- Otherwise fall back to `getenv("HOME")` + "/.cache"
- If HOME is also NULL, die("HOME not set")
- Append "/" + CACHE_DIR_NAME to form full cache directory path
- Return 0 on success, snprintf bounds check

**main(int argc, char *argv[]):**
1. Set `argv0 = argv[0]`, call `setup_sigchld()`
2. Build cache directory path via `get_cache_dir()`
3. Open directory with `opendir()`. If NULL (directory doesn't exist), exit 0 silently (per Pitfall 6)
4. Allocate `struct entry entries[MAX_ENTRIES]`
5. Iterate with `readdir()`:
   - Skip "." and ".." entries
   - Skip files starting with "." (hidden files, safety)
   - Build full path: `snprintf(path, sizeof(path), "%s/%s", cache_dir, dirent->d_name)`
   - `stat(path, &st)` to get mtime. Skip on stat failure.
   - Copy hash (d_name), mtime, and full path into entries[count]
   - Call `build_preview(path, entries[count].preview, sizeof(entries[count].preview))`
   - Increment count. Stop at MAX_ENTRIES.
6. `closedir()`
7. If count is 0, exit 0 silently (per Pitfall 6)
8. `qsort(entries, count, sizeof(struct entry), cmp_mtime_desc)` (per D-14)
9. Open dmenu: `dmenu_open("clipboard", argv + 1)` (per D-18, forward extra args)
10. Write each entry's preview to dmenu via `dmenu_write(ctx, entries[i].preview)`
11. Read selection via `dmenu_read()`. If NULL, free and exit 0.
12. Match selection back to entry: iterate entries array, `strcmp(sel, entries[i].preview)`. First match wins (newest-first order handles duplicate previews per Pitfall 4)
13. If match found, call `restore_clipboard(entries[match].path)` (per D-17)
14. Free selection, return 0

**Security considerations:**
- Symlink attack defense: stat() follows symlinks by default, but the cache directory is user-owned (~/.cache/dmenu-clipboard). Use `lstat()` instead of `stat()` to detect symlinks. If `S_ISLNK(st.st_mode)`, skip the entry. This prevents an attacker from planting symlinks in the cache dir pointing to sensitive files (per security_enforcement: symlink attacks on clipboard cache).
- PATH_MAX bounds on all path construction via snprintf
- No shell interpretation: fork+execvp/execlp only
- File read bounded by MAX_PREVIEW (no unbounded reads)
- Directory entry count bounded by MAX_ENTRIES

**Do NOT:**
- Use system() or popen()
- Read entire file contents for preview (only first MAX_PREVIEW bytes)
- Crash on empty or missing cache directory
- Use exec_wait for xclip (waitpid directly in restore_clipboard is cleaner since we need the stdin redirect)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cache directory files -> program memory | Untrusted file contents read into preview buffers |
| dmenu user input -> entry matching | User selection matched against preview strings |
| cache directory entries -> file operations | Directory entries used to construct file paths |
| cache file -> xclip stdin | File content piped to xclip for clipboard restore |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering | dmenu-clip: symlink in cache dir | mitigate | Use lstat() instead of stat(). Skip entries where S_ISLNK(st.st_mode) is true. Prevents reading arbitrary files via symlink planted in cache dir. |
| T-03-07 | Information Disclosure | dmenu-clip: preview reads sensitive file via symlink | mitigate | Same lstat() check as T-03-06. Additionally, build_preview reads only MAX_PREVIEW (80) bytes, limiting exposure. |
| T-03-08 | Denial of Service | dmenu-clip: huge cache directory | mitigate | MAX_ENTRIES cap (50) prevents unbounded memory allocation. Stop scanning after MAX_ENTRIES reached. |
| T-03-09 | Tampering | dmenu-clip: path traversal in cache filenames | accept | readdir returns actual filesystem entries. snprintf with PATH_MAX prevents buffer overflow. Cache dir is user-owned. Attacker with write access to ~/.cache already has full user compromise. |
| T-03-10 | Spoofing | dmenu-clip: duplicate preview matching wrong entry | accept | First match wins (newest-first), matching existing shell script behavior per Pitfall 4. Not a security issue -- both entries are user's own clipboard data. |
</threat_model>
