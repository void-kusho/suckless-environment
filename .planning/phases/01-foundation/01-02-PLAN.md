Implement each function:

**dmenu_open(const char *prompt, char *const extra_argv[]):**
1. Create two pipes: `to_dmenu[2]` (parent writes, child reads) and `from_dmenu[2]` (child writes, parent reads). Call `pipe(to_dmenu)` and `pipe(from_dmenu)`. On failure, call `die("pipe:")`.
2. Build argv array for dmenu: Count extra_argv entries (if non-NULL, iterate until NULL). Allocate array of size `3 + extra_count + 1` (for "dmenu", "-p", prompt, extras..., NULL). Set: `argv[0] = "dmenu"`, `argv[1] = "-p"`, `argv[2] = prompt`, then copy each extra_argv entry, then NULL terminator. Cast prompt with `(char *)` for the argv array.
3. Fork. On failure, `die("fork:")`.
4. **Child (pid == 0):** Close unused pipe ends (`to_dmenu[1]`, `from_dmenu[0]`). Redirect stdin: `dup2(to_dmenu[0], STDIN_FILENO)`. Redirect stdout: `dup2(from_dmenu[1], STDOUT_FILENO)`. Close original fd copies after dup2 (`to_dmenu[0]`, `from_dmenu[1]`). Call `execvp("dmenu", argv)`. On exec failure, `_exit(127)`.
5. **Parent:** Close unused pipe ends (`to_dmenu[0]`, `from_dmenu[1]`). Allocate DmenuCtx with `malloc(sizeof(DmenuCtx))`. If NULL, `die("malloc:")`. Set `ctx->write = fdopen(to_dmenu[1], "w")`, `ctx->read = fdopen(from_dmenu[0], "r")`, `ctx->pid = pid`. Free the argv array. Return ctx.

**dmenu_write(DmenuCtx *ctx, const char *item):**
1. `fprintf(ctx->write, "%s\n", item)`
2. `fflush(ctx->write)` -- flush immediately so dmenu can start processing

**dmenu_read(DmenuCtx *ctx):**
1. Close write end first to signal EOF to dmenu: `fclose(ctx->write)`, set `ctx->write = NULL`.
2. Read selection: allocate buffer `char *buf = malloc(1024)`. If NULL, `die("malloc:")`.
3. If `fgets(buf, 1024, ctx->read)` returns NULL (user pressed Escape / dmenu exited without selection), free buf, return NULL per D-10.
4. Strip trailing newline: `buf[strcspn(buf, "\n")] = '\0'`.
5. If result is empty string after stripping, free buf, return NULL.
6. Return buf (caller frees).

**dmenu_close(DmenuCtx *ctx):**
1. If `ctx->write` is not NULL, `fclose(ctx->write)`.
2. If `ctx->read` is not NULL, `fclose(ctx->read)`.
3. `waitpid(ctx->pid, NULL, 0)` -- reap dmenu child.
4. `free(ctx)`.

CRITICAL anti-patterns:
- NEVER use `popen()` (per D-09, violates bidirectional pipe requirement)
- NEVER use `system()` (shell injection risk)
- dmenu_read MUST close write end BEFORE reading (prevents pipe deadlock per RESEARCH.md Pitfall 1)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| dmenu_open argv -> execvp | Extra argv from callers passed through to dmenu process |
| dmenu_read pipe -> caller | Data read from dmenu child process consumed by caller |
| install.sh -> make | install.sh invokes make which compiles and installs code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | dmenu_open argv construction | mitigate | Callers construct extra_argv at compile time from trusted sources (dwm config.h keybinding args). No user-controlled input reaches argv construction. execvp does not invoke a shell. |
| T-02-02 | Tampering | dmenu_read buffer | mitigate | dmenu_read allocates fixed 1024-byte buffer and uses fgets with size limit. No buffer overflow possible. Caller frees returned pointer. |
| T-02-03 | Denial of Service | dmenu_close child reap | mitigate | dmenu_close calls waitpid on the specific dmenu child PID. If dmenu hangs, the caller blocks -- this is acceptable since dmenu is an interactive program that the user controls. |
| T-02-04 | Tampering | install.sh make invocation | accept | install.sh runs make -C utils which compiles from local source code. The user runs install.sh intentionally. No privilege escalation -- installs to ~/.local/bin without sudo. |
| T-02-05 | Information Disclosure | dmenu pipe data | accept | Data flows through kernel pipes between parent and dmenu child processes. Both run as the same user. No cross-user information disclosure. |
</threat_model>
