```makefile
	rm -f $(COMMON_OBJ) $(DMENU_OBJ)

install: all
	mkdir -p "$(DESTDIR)$(PREFIX)/bin"
	@for bin in $(BINS); do \
		cp -f "$$bin" "$(DESTDIR)$(PREFIX)/bin/$$(basename $$bin)"; \
		chmod 755 "$(DESTDIR)$(PREFIX)/bin/$$(basename $$bin)"; \
	done

uninstall:
	@for bin in $(BINS); do \
		rm -f "$(DESTDIR)$(PREFIX)/bin/$$(basename $$bin)"; \
	done

.PHONY: all clean install uninstall
```

Create `utils/common/util.c` -- a FRESH standalone implementation per D-02 (NOT a copy of slstatus/util.c). Same function signatures but independent code. The implementation must include:

1. `static void verr(const char *fmt, va_list ap)` -- prints formatted error to stderr. If format string ends with `:`, appends a space then errno string via `perror(NULL)`. Otherwise appends newline. Prepends `argv0: ` to all messages.

2. `void warn(const char *fmt, ...)` -- calls verr, continues execution.

3. `void die(const char *fmt, ...)` -- calls verr, then calls `exit(1)`.

4. `int pscanf(const char *path, const char *fmt, ...)` -- opens file at path for reading, uses `vfscanf` to scan with provided format and va_list, closes file immediately. Returns number of items matched, or -1 on fopen failure or EOF. Calls `warn("fopen '%s':", path)` on fopen failure.

5. `int exec_wait(const char *const argv[])` -- fork, child calls `execvp(argv[0], (char *const *)argv)` and `_exit(127)` on exec failure. Parent calls `waitpid(pid, &status, 0)`. Returns `WEXITSTATUS(status)` if `WIFEXITED(status)`, else -1. Calls `die("fork:")` if fork fails.

6. `void exec_detach(const char *const argv[])` -- fork, child calls `setsid()` then `execvp(argv[0], (char *const *)argv)` and `_exit(127)` on exec failure. Parent returns immediately without waiting. Calls `die("fork:")` if fork fails.

7. `void setup_sigchld(void)` -- installs SIGCHLD handler via `sigaction`. Handler calls `while (waitpid(-1, NULL, WNOHANG) > 0);` (async-signal-safe). Uses `SA_RESTART | SA_NOCLDSTOP` flags. Uses `sigemptyset` for signal mask.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| util.c exec helpers -> child process | User-provided argv arrays are passed to execvp; malicious entries could execute arbitrary programs |
| pscanf -> filesystem | Path argument opens arbitrary files for reading; attacker-controlled paths could read sensitive data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | exec_wait/exec_detach | mitigate | Use execvp with explicit argv arrays (never system()). Callers construct argv at compile time or from trusted sources. No shell interpretation occurs. |
| T-01-02 | Tampering | pscanf path argument | accept | pscanf reads sysfs paths constructed at compile time (e.g., "/sys/class/power_supply/BAT1/capacity"). No user-controlled path input in any planned utility. Low risk -- all callers are our own code. |
| T-01-03 | Denial of Service | setup_sigchld handler | mitigate | Handler calls ONLY waitpid() which is async-signal-safe per POSIX. No malloc/printf/syslog in handler. SA_RESTART flag prevents interrupted syscall failures. |
| T-01-04 | Denial of Service | exec_detach zombie leak | mitigate | setup_sigchld() installs WNOHANG reaping loop. Every utility calling exec_detach() must call setup_sigchld() at program start. Test program verifies no zombies remain. |
| T-01-05 | Information Disclosure | pscanf error messages | accept | warn() prints file paths to stderr on fopen failure. These are sysfs paths, not sensitive. Acceptable for diagnostic utility. |
</threat_model>
