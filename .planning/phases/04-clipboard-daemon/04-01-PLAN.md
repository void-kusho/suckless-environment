**Static globals (module-level state):**
- `static Display *dpy;` -- X11 display connection
- `static Window win;` -- invisible window for receiving SelectionNotify
- `static int xfixes_event_base;` -- base event number for XFixes extension
- `static Atom clipboard_atom;` -- XInternAtom("CLIPBOARD", False)
- `static Atom utf8_string_atom;` -- XInternAtom("UTF8_STRING", False)
- `static Atom xsel_data_atom;` -- XInternAtom("XSEL_DATA", False) (property for storing conversion result)
- `static int lock_fd = -1;` -- flock file descriptor
- `static volatile sig_atomic_t done = 0;` -- SIGTERM flag (per D-11)
- `static char cache_dir[PATH_MAX];` -- resolved cache directory path

**Function implementations in this order:**

1. `sigterm_handler(int sig)` -- ONLY sets `done = 1`. Per D-11. MUST be async-signal-safe: no printf, no malloc, no X calls. Cast sig to void.

2. `get_cache_dir(char *buf, size_t bufsz)` -- Identical logic to dmenu-clip/dmenu-clip.c get_cache_dir(). Check XDG_CACHE_HOME first (non-empty check per Pitfall 5 from research), fall back to $HOME/.cache. Append "/CACHE_DIR_NAME". Return -1 if snprintf truncates. This ensures both daemon and picker resolve to the same directory.

3. `acquire_lock(const char *dir)` -- Per D-10. Build lock path as `dir/.lock`. Open with O_CREAT|O_RDWR, mode 0600. Call flock(lock_fd, LOCK_EX|LOCK_NB). If flock fails with EWOULDBLOCK, die("dmenu-clipd: already running"). If flock fails for other reason, die("flock:"). flock auto-releases on process exit.

4. `fnv1a(const char *data, size_t len)` -- Per D-07. Return uint64_t. Use FNV-1a 64-bit constants: offset basis = 0xcbf29ce484222325ULL, prime = 0x100000001b3ULL. XOR each byte then multiply by prime. ~10 lines.

5. `hash_to_filename(uint64_t hash, char *buf, size_t bufsz)` -- Format hash as 16-char hex string using snprintf(buf, bufsz, "%016llx", (unsigned long long)hash).

6. `is_whitespace_only(const char *s, size_t len)` -- Return 1 if string is empty or contains only spaces, tabs, newlines, carriage returns. Simple loop checking each character. Per Pitfall 5 from research.

7. `setup_x11(void)` -- Per D-01, D-02, D-03:
   - XOpenDisplay(NULL), die if NULL
   - XFixesQueryExtension(dpy, &evt_base, &err_base), die if not available
   - Store xfixes_event_base = evt_base
   - Create invisible window: XCreateSimpleWindow(dpy, DefaultRootWindow(dpy), 0, 0, 1, 1, 0, 0, 0)
   - XInternAtom for "CLIPBOARD", "UTF8_STRING", "XSEL_DATA"
   - xa_string = XA_STRING (predefined in Xatom.h, use as fallback)
   - XFixesSelectSelectionInput(dpy, win, clipboard_atom, XFixesSetSelectionOwnerNotifyMask)
   - XFlush(dpy) to ensure registration is sent
   CRITICAL: Only register for clipboard_atom (D-01), NOT XA_PRIMARY (D-02).

8. `get_clipboard_text(size_t *out_len)` -- Per D-04, D-05:
   - Call XConvertSelection(dpy, clipboard_atom, utf8_string_atom, xsel_data_atom, win, CurrentTime)
   - XFlush(dpy)
   - Wait for SelectionNotify using a bounded loop: up to 50 iterations of XCheckTypedWindowEvent(dpy, win, SelectionNotify, &ev) with usleep(10000) (10ms) between checks = 500ms max wait. If no event arrives, return NULL.
   - If ev.xselection.property == None: UTF8_STRING failed. Try STRING fallback (per D-05): repeat XConvertSelection with XA_STRING, wait again with same bounded loop. If both fail, return NULL.
   - On success: XGetWindowProperty(dpy, win, xsel_data_atom, 0, LONG_MAX, True, AnyPropertyType, &actual_type, &actual_format, &nitems, &bytes_after, &prop_data). The True parameter deletes property after read (ICCCM).
   - If prop_data and nitems > 0: malloc(nitems + 1), memcpy, null-terminate, set *out_len = nitems.
   - XFree(prop_data) if non-NULL.
   - XDeleteProperty(dpy, win, xsel_data_atom) if conversion failed (cleanup).
   - Return malloc'd string (caller frees) or NULL.

9. `store_entry(const char *text, size_t len)` -- Per D-06, D-07, D-08, D-09:
   - Compute hash = fnv1a(text, len)
   - hash_to_filename(hash, hashname, sizeof(hashname))
   - Build path: snprintf(path, sizeof(path), "%s/%s", cache_dir, hashname)
   - Check snprintf bounds (skip if truncated)
   - Open file O_WRONLY|O_CREAT|O_TRUNC, mode 0600. If open fails, warn and return.
   - write(fd, text, len). Close fd.
   - Note: writing to an existing hash file (same content copied again) updates mtime, making it appear newest in dmenu-clip. This is correct behavior matching the shell script.
   - Call prune_old_entries() after writing.

10. `prune_old_entries(void)` -- Per D-08:
    - opendir(cache_dir). If NULL, return.
    - Scan entries: skip d_name[0]=='.' (skips .lock and hidden files), stat each file, skip non-regular, collect into array of {name, mtime} structs. Size array at MAX_ENTRIES + 20 to handle overflow.
    - closedir.
    - If count <= MAX_ENTRIES, return (nothing to prune).
    - qsort by mtime ascending (oldest first).
    - unlink the first (count - MAX_ENTRIES) entries.

11. `handle_event(XEvent *ev)` -- Per D-03, D-04:
    - Check ev->type == xfixes_event_base + XFixesSelectionNotify
    - If yes: call get_clipboard_text(&len)
    - If text is non-NULL, len > 0, and !is_whitespace_only(text, len): call store_entry(text, len)
    - free(text)
    - Silently ignore all other event types.

12. `run_event_loop(void)` -- Per D-03, D-13:
    - int xfd = ConnectionNumber(dpy);
    - Main loop: while (!done) {
        - Drain buffered events first (Pitfall 2): while (XPending(dpy) > 0) { XNextEvent(dpy, &ev); handle_event(&ev); if (done) return; }
        - Set up select(): FD_ZERO(&fds); FD_SET(xfd, &fds); tv.tv_sec=1; tv.tv_usec=0;
        - select(xfd + 1, &fds, NULL, NULL, &tv);
        - On EINTR (signal): loop continues, done flag checked at top
        - On timeout: loop continues, done flag checked at top
        - On data ready: loop continues, XPending will find events
      }

13. `main(int argc, char *argv[])` -- Per D-10, D-11, D-12:
    - (void)argc; argv0 = argv[0];
    - get_cache_dir(cache_dir, sizeof(cache_dir)), die if fails
    - mkdir(cache_dir, 0700) -- create if missing. Check errno != EEXIST on failure, die otherwise.
    - acquire_lock(cache_dir) -- per D-10
    - Set up SIGTERM and SIGINT handlers via sigaction (NOT signal()). sa_flags = 0 (do NOT set SA_RESTART -- we want select() interrupted). Per D-11.
    - setup_x11() -- per D-03
    - run_event_loop() -- per D-13
    - XDestroyWindow(dpy, win); XCloseDisplay(dpy) -- per D-12
    - close(lock_fd) if lock_fd >= 0 -- per D-12. flock released automatically.
    - return 0;

**Critical implementation notes:**
- Do NOT use setup_sigchld() from util.h -- this daemon does not fork child processes.
- Do NOT link dmenu.h/dmenu.o -- this daemon never interacts with dmenu.
- The XA_STRING atom is predefined in X11/Xatom.h as a macro, do NOT XInternAtom it. Store it in a static Atom variable for consistent use alongside utf8_string_atom.
- All file paths constructed with snprintf and bounds-checked. No unchecked string operations.
- Lock file path uses ".lock" prefix (hidden file) so dmenu-clip skips it (d_name[0] == '.').
- Cache directory mode 0700 for access control (ASVS V4).
- Lock file mode 0600 for access control.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| X11 clipboard -> daemon | Untrusted content from any X11 application enters via XGetWindowProperty |
| daemon -> filesystem | Daemon writes files to user-owned cache directory |
| filesystem -> dmenu-clip | dmenu-clip reads files the daemon wrote (cross-process trust) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | Cache directory | mitigate | mkdir(cache_dir, 0700) restricts access to owner only. Lock file created with mode 0600. |
| T-04-02 | Tampering | Symlink in cache dir | accept | Daemon writes to known hash filenames with O_CREAT|O_TRUNC. Does not follow symlinks when reading (that is dmenu-clip's concern, already mitigated with lstat in Phase 3). If an attacker plants a symlink with a hash filename, the daemon would follow it -- but attacker needs write access to 0700 directory first, which means they already have user-level access. Low residual risk. |
| T-04-03 | Information Disclosure | Clipboard content in plaintext | accept | Inherent to all clipboard managers. Cache dir mode 0700 limits exposure. Sensitive data (passwords) in clipboard is a user behavior concern, not a daemon concern. |
| T-04-04 | Denial of Service | Rapid clipboard changes | mitigate | Event-driven architecture processes one event at a time. No amplification. LRU pruning bounds cache at MAX_ENTRIES files. select() timeout prevents CPU spin. |
| T-04-05 | Tampering | Signal handler safety | mitigate | SIGTERM handler ONLY sets `done = 1` (volatile sig_atomic_t). No malloc, printf, or X11 calls in handler. All cleanup in main flow after event loop exits. |
| T-04-06 | Spoofing | flock bypass | accept | flock() is advisory. A malicious process could ignore the lock and write to cache. However, this requires user-level access and intentional bypass. PID files are worse (stale on crash). flock is the standard POSIX mechanism. |
| T-04-07 | Information Disclosure | X11 clipboard data injection | mitigate | Validate clipboard is text (UTF8_STRING or STRING target only, per D-05). Skip binary/image data. Bound file writes to actual content length. No shell interpretation of content. |
</threat_model>
