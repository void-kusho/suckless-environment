# Requirements: Suckless Environment v2

**Defined:** 2026-04-09
**Core Value:** Every user-facing utility is a fast, safe, native C program that works reliably without runtime dependencies on shell interpreters.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Foundation

- [ ] **FOUND-01**: Shared utility library (die, warn, pscanf, exec_wait, exec_detach) modeled on slstatus/util.c
- [ ] **FOUND-02**: Shared dmenu pipe helpers (dmenu_open, dmenu_select) using fork+dup2+exec pattern
- [ ] **FOUND-03**: Per-utility Makefile with config.mk convention matching dwm/dmenu/slstatus
- [ ] **FOUND-04**: Updated install.sh compiles all C utilities and installs binaries to ~/.local/bin

### Clipboard Daemon

- [ ] **CLIPD-01**: Clipboard daemon monitors X11 clipboard changes via XFixes events (no polling)
- [ ] **CLIPD-02**: Clipboard daemon stores text-only entries with FNV-1a hash dedup in ~/.cache/dmenu-clipboard/
- [ ] **CLIPD-03**: Clipboard daemon prunes entries beyond 50 using LRU (oldest mtime removed)
- [ ] **CLIPD-04**: Clipboard daemon enforces single instance via flock
- [ ] **CLIPD-05**: Clipboard daemon shuts down gracefully on SIGTERM

### Clipboard Picker

- [ ] **CLIP-01**: User can browse clipboard history newest-first via dmenu
- [ ] **CLIP-02**: Each entry shows 80-char truncated preview with newlines replaced by spaces
- [ ] **CLIP-03**: User selecting an entry copies full content to clipboard via xclip
- [ ] **CLIP-04**: dmenu-clip passes extra arguments through to dmenu

### Session Manager

- [ ] **SESS-01**: User can lock screen via betterlockscreen with blur 0.8 and DISPLAY export
- [ ] **SESS-02**: Lock prevents duplicate betterlockscreen instances (pgrep check)
- [ ] **SESS-03**: User can logout with confirmation dialog (kills slstatus, dmenu-clipd, dwm)
- [ ] **SESS-04**: User can reboot with confirmation dialog (systemctl reboot)
- [ ] **SESS-05**: User can shutdown with confirmation dialog (systemctl poweroff)

### Power Profiles

- [ ] **POWER-01**: User can see current power profile in dmenu prompt
- [ ] **POWER-02**: User can select from available profiles (performance/balanced/power-saver)
- [ ] **POWER-03**: Selected profile is applied via powerprofilesctl set
- [ ] **POWER-04**: dmenu-cpupower passes extra arguments through to dmenu

### Battery Monitor

- [ ] **BATT-01**: One-shot read of /sys/class/power_supply/BAT1/ for capacity and status
- [ ] **BATT-02**: Sends dunst notification when capacity ≤20% and status is Discharging
- [ ] **BATT-03**: Uses state file (/tmp/battery-notified) to prevent re-alert spam
- [ ] **BATT-04**: Clears state file when battery rises above threshold or starts charging

### Screenshot

- [ ] **SCRN-01**: Captures selected area via maim -s piped to xclip -selection clipboard -t image/png
- [ ] **SCRN-02**: Sends dunst notification "Image copied to clipboard" on successful capture
- [ ] **SCRN-03**: Exits silently when user cancels selection (no false notification)

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Clipboard Daemon Improvements

- **CLIPD-06**: Full SelectionRequest serving (TARGETS + UTF8_STRING) to prevent data loss on app close
- **CLIPD-07**: Atomic file writes (write to .tmp, then rename())
- **CLIPD-08**: INCR protocol support for clipboard entries >256KB

### Battery Monitor Improvements

- **BATT-05**: Two-tier alerts (warning at 20%, critical at 5% with persistent notification)
- **BATT-06**: systemd .timer + .service unit files for periodic execution
- **BATT-07**: Robust battery device detection via sysfs scanning (instead of hardcoded BAT1)

### Screenshot Improvements

- **SCRN-04**: Multiple capture modes (area, fullscreen, active window) via argument flags

### Session Manager Improvements

- **SESS-06**: Suspend/hibernate support

## Out of Scope

| Feature | Reason |
|---------|--------|
| Wayland support | X11 only — explicit project boundary |
| dmenu-clip direct Xlib clipboard (no xclip) | Complex SelectionRequest event loop; xclip dependency acceptable for v1 |
| libnotify for notifications | exec notify-send is simpler, consistent with suckless exec-everything philosophy |
| sd-bus direct D-Bus calls for notifications | Adds complexity; exec notify-send sufficient for v1 |
| Multi-monitor betterlockscreen config | Single display target per PROJECT.md |
| Interactive screenshot tools (annotation, crop UI) | Just capture and notify |
| dwm/st/dmenu/slstatus source modifications | Separate concern |
